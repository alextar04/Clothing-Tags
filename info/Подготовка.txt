Приложение, сохраняющая информацию о бирках с одежды.

Идея:
После покупки одежды, нет необходимости сохранять бирки с информацией о материалах, температурных режимах и так далее. Бирки могут быть неприятными для тела и их требуется удалить, а затем они теряются. 

Технологии:
Swift 5, iOS 10
Глобально - приложение использует MVVM - паттерн
Дополнительно - паттерны для соц.cетей и тд
Данные хранятся в приложении при помощи CoreData
Асинхронность при запросах в БД при помощи GCD (GrandCentralDispatch)
Передача данных в приложении приоритетно с помощью swiftRX
Запросы через API соцсетей - через Alamofire
Тесты ??

Функционал:

- Стартовая страница со всеми категориями одежд из БД:
	- Добавить категорию на верхнем баре
		- Название и картинка категории 
	- Cвап категории влево - удалить (красный маленький бокс) или модифицировать (желтый маленький бокс)

- Страница категории:
	- Добавить одежду через плюс на верхнем баре
		- Cтраница
		- QR код, затем перебрасывающий на страницу
	- Cвап одежды влево - удалить (красный маленький бокс) или модифицировать (желтый маленький бокс)

- Страница одежды:
	- На баре - кнопка ,соединяющая:
		- Редактирования и удаления
		- Сохранить в заметки (снизу открывается предложение в VK, FB, Yandex)
			- Страницы идентификации для каждой соцсети (либо своя, либо через селениум). Этого достаточно для публикации
	- Напоминание: Хочу постирать!
		- Страница, выбирающая параметры оповещения. API - apple для оповещений

- Нижний бар:
	- Бирки
	- Ожидающие стирки
